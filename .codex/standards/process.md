# Process Standards

This document defines the development workflow, roles, and procedures.

## Branching Model

<!-- BEGIN_BRANCHING -->
**Trunk-Based Development**

- `main` is the single source of truth
- Feature branches are short-lived (< 2 days)
- All changes require pull requests
- Squash merge to maintain clean history
<!-- END_BRANCHING -->

## Release Process

<!-- BEGIN_RELEASE -->
| Phase | Activities | Owner |
|-------|------------|-------|
| Planning | Requirements, Design | Contributor |
| Implementation | Code, Tests, Docs | Contributor |
| Validation | CI checks | Automated |
| Review | PR approval | Reviewer |
| Release | Tag, Deploy | Maintainer |
<!-- END_RELEASE -->

## Checklist

<!-- BEGIN_CHECKLIST -->
| Phase | Mandatory Artifacts | Owner |
|-------|---------------------|-------|
| Planning | Requirements documented | Contributor |
| Implementation | Code + Tests | Contributor |
| Validation | All checks passing | CI |
| Review | At least 1 approval | Reviewer |
<!-- END_CHECKLIST -->

## Roles

- **Contributor**: Submits code and documentation changes
- **Reviewer**: Reviews PRs for compliance and quality
- **Maintainer**: Merges PRs and manages releases
